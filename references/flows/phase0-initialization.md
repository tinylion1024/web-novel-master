# 第0步：初始化与偏好加载

**每次触发SKILL时首先执行：**

---

## 1. 偏好加载 + 中断续写检测（并行）

同时执行两项检查：

**A. 读取用户偏好**
- 检查 `user-preferences.json` 是否存在
- 存在 → 加载偏好数据，用于后续选项排序、推荐标记、个性化欢迎语
- 不存在 → 使用默认值，首次交互结束后自动创建

**B. 扫描未完成项目**
- 扫描 `./web-novels/` 目录，查找含 `03-写作计划.json` 且 `status` 为 `"in_progress"` 或 `"validating"` 的项目
- 如有偏好数据，展示个性化欢迎语：
  > "欢迎回来！基于你之前的创作习惯，我为你做了一些推荐（标记为⭐的选项）。"

偏好数据结构与更新规则详见 → [shared-infrastructure.md](shared-infrastructure.md)

---

## 2. 中断续写处理

如果检测到未完成项目：
- 展示项目信息：小说名称、完成进度（X/Y 章已完成）
- 提供选项：
  - "继续上次创作" → 直接进入第三阶段，从中断章节继续
  - "开始新作品" → 进入第一层问答

如果无未完成项目 → 进入步骤3

---

## 3. 快捷入口判断

分析用户的首次输入是否已包含丰富的创作信息（题材、爽点类型、金手指等关键要素）

- **信息充足** → 展示提取结果，提供三个选项：
  - "是的，直接开始规划" → 跳过问答，进入第二阶段
  - "先展示提取的信息让我确认/修改" → 展示配置供确认后进入规划
  - "不，我还是想走一遍问答流程" → 进入第一层问答
- **信息不足** → 直接进入第一层问答

---

## 4. 确认进入

确保 `./web-novels/` 目录存在（如不存在则创建）。

向用户确认进入创作流程：

```
🎯 爆款网文创作模式启动！

请选择：
- 【快速开始】直接告诉我你想写什么小说，我来帮你规划
- 【详细问答】一步步引导你完成所有创作配置
```
