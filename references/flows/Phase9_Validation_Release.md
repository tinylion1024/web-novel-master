# Phase 9 – Validation & Release

**目标**：自动校验章节质量、修复不合格章节、生成发布报告

---

## 9.1 初始化校验

读取 `03-写作计划.json`，`status` 更新为 `"validating"`

---

## 9.2 逐章检查

对每一章执行：

1. **文件存在性** — 检查 `filePath` 是否存在
2. **字数检查** — `python scripts/check_chapter_wordcount.py <文件路径>`
3. **爽点检查**：
   - 本章是否有核心爽点？
   - 爽点密度是否达标（每2000字≥1个爽点）？
   - 是否有金句？
4. **情绪曲线检查** — 是否有"压抑→释放"弧线？
5. **人物一致性检查**：
   - 对话语气是否符合性格？
   - 主角能力值/状态是否与上章衔接？
6. **伏笔检查**：
   - 规划回收的伏笔是否已回收？
   - 本章埋下的伏笔是否已记录？
7. **更新 JSON** — 写入 `wordCount`、`wordCountPass`、校验结果

---

## 9.3 校验维度

| 维度 | 目标 | 不合格处理 |
|-----|------|----------|
| **字数** | 2000-3000字/章 | 扩充或压缩 |
| **爽点密度** | 每2000字≥1个爽点 | 增强爽感描写 |
| **金句** | 每章至少1个金句 | 强化结尾/高潮 |
| **开头钩子** | 前20%有即时爽点 | 重写开头 |
| **情绪曲线** | 有压抑→释放弧线 | 调整节奏 |
| **人物语气** | 对话符合性格设定 | 调整对话 |
| **伏笔回收** | 规划回收的伏笔需兑现 | 补充伏笔回收 |

---

## 9.4 自动修复（最多3轮）

对每个不合格章节：

1. `status` 设为 `"failed"`，`retryCount` 加 1
2. 执行「逐章创作流程」（Phase 6 第6.2节）
3. 重新检查字数和爽点
4. 更新 JSON

**循环规则**：
- 最多 3 轮校验-修复循环
- 超过3次仍不合格 → 保留记录但不阻塞流程

---

## 9.5 汇总结果

- **全部通过** → `status` 设为 `"completed"`，进入步骤 9.6
- **有不合格但已重试** → 进入步骤 9.6
