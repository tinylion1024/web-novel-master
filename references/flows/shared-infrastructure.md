# 共享机制

本文件定义跨阶段共享的机制和规则。

---

## 爆款网文三大法则

1. **爽点驱动** - 每章必须有爽点，读者追更是为了"爽"
2. **情绪波动** - 有憋有放，张弛有度，虐后必爽
3. **金句记忆** - 每章必须有让读者想截图的金句

## 用户偏好系统

### 存储文件

`user-preferences.json`（项目根目录，首次使用后自动创建）

### 数据结构

```json
{
  "version": 2,
  "updatedAt": "2026-04-18",
  "preferences": {
    "favoriteGenres": [],
    "preferredGoldenFinger": "",
    "preferredProtagonist": "",
    "preferredCoreTropes": "",
    "typicalChapterCount": null,
    "styleReferences": [],
    "dislikes": [],
    "creationHistory": []
  }
}
```

### 偏好更新规则

| 时机 | 行为 |
|------|------|
| 每完成一层问答 | 静默将本层回答同步到偏好文件（追加/更新，不删除历史） |
| 用户说"记住我的偏好" | 保存当前所有配置到偏好 |
| 用户说"忘记XX偏好" | 清除指定维度的偏好 |
| 用户说"重置偏好" | 清空所有偏好数据 |
| 一部长篇创作完成 | 将作品信息追加到 `creationHistory` |

### 偏好如何影响交互

1. **启动欢迎语**：有偏好时显示"欢迎回来！" + 个性化提示
2. **选项排序**：Q1中将 `favoriteGenres` 匹配项排前面
3. **常用标记**：常用选项标记"你的常用"/"上次选择"
4. **需求报告**：结合偏好给个性化建议
5. **随机生成**：优先从偏好范围内随机选取，保持一致性

### 错误恢复

- **回退修改**：用户随时可说"回到QX"、"修改XX"，AI 回退到指定问题重新询问
- **中途暂存**：通过 `03-写作计划.json` 实现自动暂存；下次触发SKILL时 Phase 0 自动检测未完成项目，询问"继续上次的《XXX》？"
- **偏好文件损坏**：JSON解析失败时忽略偏好，使用默认值，并在后台修复文件

## 写作计划系统

### 存储文件

`03-写作计划.json`（项目文件夹内，Phase 2 创建）

### 作用

- **进度跟踪**：记录每章创作状态（pending/in_progress/completed/failed）
- **写作模式**：记录用户选择的写作模式（serial/subagent-parallel/agent-teams）
- **名场面跟踪**：记录每章的名场面类型和描述
- **中断续写**：Phase 0 读取 JSON 检测未完成项目，支持从断点继续
- **校验依据**：Phase 4 基于 JSON 校验章节完成度、字数、爽点

### 与大纲的关系

- `00-大纲.md`：章节规划（核心事件、爽点设计、悬念钩子）+ 章节摘要（连贯性参考）
- `03-写作计划.json`：章节状态、字数、爽点标记、重试次数（机器可读的进度跟踪）
- Phase 3 创作每章时必须读取 `00-大纲.md` 中对应章节的规划信息，作为创作依据
- 两者严格对应：JSON 中的 `chapterNumber` 和 `title` 必须与大纲中的章节规划一致

### JSON 损坏处理

- JSON 解析失败时：提示用户，尝试从大纲的章节摘要区推断完成进度
- 章节状态丢失时：通过文件存在性和字数脚本重建状态

## 字数检查脚本

使用 `scripts/check_chapter_wordcount.py` 检查章节字数：

```bash
# 检查单个章节
python scripts/check_chapter_wordcount.py ./web-novels/项目文件夹/第01章.md

# 检查所有章节
python scripts/check_chapter_wordcount.py --all ./web-novels/项目文件夹/

# 自定义最小字数
python scripts/check_chapter_wordcount.py ./web-novels/项目文件夹/第01章.md 2500
```

### 使用场景

| 阶段 | 用途 |
|------|------|
| Phase 3（逐章创作） | 撰写后检查单章字数，低于2000字必须扩充 |
| Phase 4（自动校验） | 批量检查所有章节字数，不合格章节自动重写 |

低于2000字的章节必须使用 [爽点设计指南.md](../guides/爽点设计指南.md) 的扩充技巧进行扩充。

## 爆款网文速查表

### 爽点类型速查

| 爽点类型 | 适用场景 | 关键词 |
|---------|---------|-------|
| 打脸爽 | 退婚/羞辱/质疑 | "你不是说我废物吗？" |
| 升级爽 | 突破/获得传承 | "境界突破！" |
| 装逼爽 | 被逼展示/身份反差 | "不是我想装..." |
| 甜宠爽 | 吃醋/护短/撒糖 | "以后离他远点" |
| 复仇爽 | 清算/打脸仇人 | "该还了" |
| 燃点爽 | 守护/热血/战斗 | "谁都不能动..." |

### 金句公式

```
打脸金句："你说的XX...好像有点短？"
装逼金句："不是我想装，是你们逼我的。"
燃点金句："谁都不能动我在乎的人。"
甜点金句："以后离他远点，听见没？"
```

### 开篇公式

```
第1章 = 觉醒爽点 + 大钩子

退婚流：废物开局 → 退婚羞辱 → 意外觉醒
系统流：濒死危机 → 系统激活 → 新手任务
重生流：死亡/失败 → 重生节点 → 先知优势
```

## 项目目录结构

```
web-novels/                          # 项目根目录
└── {timestamp}-{小说名称}/
    ├── 00-大纲.md                    # 故事大纲 + 章节规划
    ├── 01-人物档案.md                # 主角/女主/反派档案
    ├── 03-写作计划.json              # 写作进度状态
    └── 第XX章-xxx.md                # 各章正文
```
