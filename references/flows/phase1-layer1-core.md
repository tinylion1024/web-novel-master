# 第一阶段·第一层：核心定位（必答，4问）

> 本层问题为必答项，确定网文的核心爽点框架。每个问题提供热门选项 + 自由输入。

---

## Q1：网文类型与赛道

使用 `AskUserQuestion` 工具询问：

```
Question: 你想要创作什么类型的爆款网文？
Options:
- 【都市爽文】神医/兵王/总裁/保镖/上门女婿
- 【玄幻修仙】废材逆袭/系统觉醒/宗门崛起/炼丹
- 【穿越重生】历史穿越/末世囤货/仙帝重生/校园重生
- 【甜宠言情】霸道总裁/校园甜文/病娇/马甲掉落
- 【悬疑惊悚】盗墓探险/灵异/刑侦/直播算命
- 【脑洞文】系统流/直播流/反派流/无敌流
- ⭐自由描述（我有明确的想法）
```

> **偏好集成**：如果 `favoriteGenres` 中有匹配项，将对应选项排在最前面并标记⭐。

**记录**：`网文类型 = [用户选择]`
**记录**：`类型细分 = [如：都市-神医/兵王/总裁]`

## 第一层完成 → 摘要展示与过渡

Q1-Q4 全部完成后，执行以下两步：

### 1. 展示已明确信息

向用户展示：

```markdown
## 已收集核心信息

🎯 **类型**：[网文类型] — [类型细分]
🔮 **金手指**：[金手指类型] — [金手指名称]
💥 **核心爽点**：[核心爽点]
🧑 **主角**：[主角数量] | [主角身份] | [主角性格]
```

### 2. 静默更新偏好

将第一层的回答同步到 `user-preferences.json`（不打扰用户）。

偏好数据结构与更新规则详见 → [shared-infrastructure.md](shared-infrastructure.md)

### 3. 过渡提示

一句话引导进入深度定制：

> "核心定位完成！接下来是深度设定（世界观、女主设定、反派设计、名场面规划等）。准备好了吗？"

→ 进入 **第二层：深度定制与创作规格**，详见 [phase1-layer2-customize.md](phase1-layer2-customize.md)
