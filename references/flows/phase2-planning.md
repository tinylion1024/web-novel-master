# 第二阶段：规划 + 二次确认

执行以下步骤：

---

## 1. 创建项目文件夹

`./web-novels/{YYYYMMDD-HHmmss}-{小说名称}/`（相对当前工作目录）

---

## 2. 生成大纲

创建 `00-大纲.md`，包含以下内容：

### 2.1 基本信息

```markdown
# [小说名称]

## 基本信息
- **类型**：[网文类型]
- **金手指**：[金手指名称]
- **核心爽点**：[核心爽点类型]
- **读者定位**：[目标读者]
- **章节数**：[X]章
```

### 2.2 核心设定

```markdown
## 核心设定

### 金手指设计
- 金手指类型：
- 激活条件：
- 使用限制：
- 升级方式：

### 等级/力量体系（如适用）
- 等级1：
- 等级2：
- ...

### 势力分布
- 主角阵营：
- 主要反派势力：
- 中立势力：
```

### 2.3 人物档案

```markdown
## 人物档案

### 主角
- 姓名：
- 身份：[初始身份]
- 性格：[主角性格]
- 金手指：[金手指]
- 成长阶段：[废柴→小成→大成→巅峰]

### 女主（如有）
- 姓名：
- 类型：[女主类型]
- 与主角关系：
- 身份背景：

### 主要反派
- 姓名/代号：
- 身份：
- 特点：[反派特点]
- 实力：
```

### 2.4 名场面时间轴

```markdown
## 名场面时间轴

| 章节 | 名场面类型 | 名场面描述 | 预期效果 |
|------|----------|-----------|---------|
| 第1章 | [类型] | [描述] | [效果] |
| 第3章 | [类型] | [描述] | [效果] |
| 第5章 | [类型] | [描述] | [效果] |
| ... | ... | ... | ... |
```

### 2.5 章节规划（每章核心事件）

```markdown
## 章节规划

### 第1章：[章节名]
- **核心事件**：
- **爽点设计**：
- **承接**：无（首章）
- **悬念钩子**：

### 第2章：[章节名]
- **核心事件**：
- **爽点设计**：
- **承接**：[上一章结尾]
- **悬念钩子**：

...（继续规划所有章节）
```

---

## 3. 生成人物档案（详细版）

创建 `01-人物档案.md`，使用 [character-template.md](../guides/character-template.md) 模板，创建主角、女主、配角、反派档案。

---

## 4. 生成写作计划

创建 `03-写作计划.json`，基于大纲内容填充，结构如下：

```json
{
  "version": 2,
  "novelName": "[小说名称]",
  "projectPath": "./web-novels/{timestamp}-[小说名称]",
  "totalChapters": [章节数],
  "minWordsPerChapter": 2000,
  "maxWordsPerChapter": 3000,
  "createdAt": "[ISO时间]",
  "updatedAt": "[ISO时间]",
  "status": "planning",
  "writingMode": "[serial|subagent-parallel|agent-teams]",
  "coreSetting": {
    "genre": "[网文类型]",
    "goldenFinger": "[金手指类型]",
    "coreTropes": "[核心爽点]"
  },
  "chapters": [
    {
      "chapterNumber": 1,
      "title": "[章节标题]",
      "filePath": "第01章.md",
      "status": "pending",
      "wordCount": null,
      "wordCountPass": null,
      "famousScene": "[名场面类型]",
      "famousSceneDesc": "[名场面描述]",
      "retryCount": 0
    }
  ]
}
```

---

## 5. 展示规划摘要 + 写作模式选择（一次确认）

向用户展示规划摘要并同时选择写作模式：

```markdown
## 《[小说名称]》创作规划确认

📊 **规模**: [X] 章 | 每章 2000-3000 字 | 预计总字数 [X-X] 万字

🧑 **主角**: [主角名称] | [身份] | [性格]
👥 **女主**: [女主名称] | [类型]
😈 **反派**: [反派类型] | [特点]

⭐ **名场面**: [X] 个（每 X 章 1 个）
- 第1章: [名场面类型] - [描述]
- 第3章: [名场面类型] - [描述]
- ...

🔥 **核心爽点**: [核心爽点]
🔮 **金手指**: [金手指]

---
**请选择写作模式：**
- 【串行】主 Agent 逐章写，稳定可靠（适合短中篇）⭐
- 【并行】子 Agent 分批并行，追求速度（适合中长篇）
- 【Teams】Claude Code 多 Agent 协作（适合大型长篇，需手动开启）
```

使用 `AskUserQuestion` 一次确认：
```
Question: 确认以上规划并选择写作模式
Options:
- 确认，开始创作（串行模式）⭐
- 确认，使用并行模式
- 确认，使用 Agent Teams 模式
- 我想修改某些设置
```

用户选择后：
- 用户选择任意创作模式 → 更新 `03-写作计划.json` 的 `writingMode` 字段，`status` 设为 `"in_progress"`
- 用户选择修改 → 回到对应的问答阶段修改

→ 进入第三阶段：疯狂创作 → 详见 [phase3-writing.md](phase3-writing.md)
