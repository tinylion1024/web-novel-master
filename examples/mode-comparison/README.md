# 四模式对照样例

同一输入：`旧物修复师在火灾后回到三年前，触摸旧物会看到记忆，但会遗忘自己的往事。`

| 模式 | 首次需要的输入 | 首轮交付物 | 适合的用户决策 |
|---|---|---|---|
| Fast | 题材、主角、一个冲突 | 首章方向、简要人设、快速大纲 | 先判断是否想继续写 |
| Professional | 题材、人物目标、能力限制、情感/爽点方向 | 世界与人设、大纲、章节任务、校验路径 | 单人系统化创作 |
| Industrial | 加上市场假设、分工、质量阈值和交付节奏 | 模块大纲、角色标准、任务拆分、QC 流程 | 团队或长篇协作 |
| Instant | 一句话创意与明确篇幅 | 一次性候选书名、简介、大纲和草稿 | 快速验证创意，不替代后续编辑 |

## 选择方法

1. 只想验证题材是否有感觉，先用 Fast 或 Instant。
2. 已经确定要完成中长篇，使用 Professional，并建立 [连续创作模板](../../references/guides/Serial-Continuity-Template.md)。
3. 有明确分工和内容负责人时，再使用 Industrial；不要为了“多 Agent”而跳过共同的世界观与验收标准。

四种模式都需要人工核对原创性、事实资料和平台规则。
