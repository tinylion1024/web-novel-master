# 贡献指南

感谢你帮助完善 Web Novel Master。这个项目欢迎小而清晰、可验证的贡献。

## 可以贡献什么

- 修复流程、链接、脚本或文档错误。
- 增加原创的题材模板、章节规划样例或质量检查建议。
- 补充可核验的数据来源和适用范围。
- 改进安装、首次体验和无障碍阅读。

## 提交前要求

1. 不提交未获授权的小说正文、作者私稿或可识别的模仿文本。
2. 使用原创人名、地点、机构和道具；在新增样例中写明命名来源或命名原则。
3. 数据类内容写清来源、发布时间和访问日期；无法核验时明确标记为待验证。
4. 保持改动聚焦，一个 Pull Request 只解决一个明确问题。

## 本地检查

```bash
python3 -m unittest discover -s tests -v
python3 scripts/validate_docs.py
```

如变更了项目管理工具，请至少运行一次相关命令并在 Pull Request 中说明结果。

## 提交流程

1. 先搜索已有 Issue，避免重复工作。
2. 对较大改动先创建 Issue，写清用户问题、预计改动和验收方式。
3. 从 `main` 创建分支并完成修改。
4. 使用 Pull Request 模板说明行为变化、示例和校验结果。

## 讨论与行为

请遵守 [行为准则](./CODE_OF_CONDUCT.md)。对创作建议保持尊重，聚焦可复用的方法，而不是评价具体作者或读者群体。
