# 共享基础设施

本文件定义跨阶段共享的机制、规则和数据结构。

---

## 爆款网文四法则

1. **爽点驱动** — 每章必须有爽点，读者追更是为了"爽"
2. **情绪波动** — 有憋有放，张弛有度，虐后必爽
3. **金句记忆** — 每章必须有让读者想截图的金句
4. **名场面** — 每3-5章必须有让人印象深刻的名场面

## 用户偏好系统

### 存储文件

`user-preferences.json`（项目根目录，首次使用后自动创建）

### 数据结构

```json
{
  "version": 2,
  "updatedAt": "2026-04-18",
  "preferences": {
    "favoriteGenres": [],
    "preferredGoldenFinger": "",
    "preferredProtagonist": "",
    "preferredCoreTropes": "",
    "typicalChapterCount": null,
    "styleReferences": [],
    "dislikes": [],
    "creationHistory": []
  }
}
```

### 偏好更新规则

| 时机 | 行为 |
|------|------|
| 每完成一个 Phase | 静默将本 Phase 结果同步到偏好文件 |
| 用户说"记住我的偏好" | 保存当前所有配置到偏好 |
| 用户说"忘记XX偏好" | 清除指定维度的偏好 |
| 用户说"重置偏好" | 清空所有偏好数据 |
| 一部作品创作完成 | 将作品信息追加到 `creationHistory` |

### 偏好如何影响交互

1. **启动欢迎语**：有偏好时显示"欢迎回来！" + 个性化提示
2. **选项排序**：将匹配项排前面并标记⭐
3. **常用标记**：常用选项标记"你的常用"/"上次选择"
4. **随机生成**：优先从偏好范围内随机选取

### 错误恢复

- **回退修改**：用户随时可说"回到QX"、"修改XX"
- **中途暂存**：通过 `03-写作计划.json` 实现自动暂存
- **偏好文件损坏**：忽略偏好，使用默认值

## 写作计划系统

### 存储文件

`03-写作计划.json`（项目文件夹内，在大纲规划阶段创建）

### 作用

- **进度跟踪**：记录每章创作状态（pending/in_progress/completed/failed）
- **写作模式**：记录用户选择的写作模式
- **名场面跟踪**：记录每章的名场面类型和描述
- **中断续写**：初始化阶段读取 JSON 检测未完成项目
- **校验依据**：校验阶段基于 JSON 校验章节完成度

### JSON Schema

```json
{
  "version": 3,
  "novelName": "[小说名称]",
  "projectPath": "./web-novels/{timestamp}-[小说名称]",
  "totalChapters": [章节数],
  "minWordsPerChapter": 2000,
  "maxWordsPerChapter": 3000,
  "createdAt": "[ISO时间]",
  "updatedAt": "[ISO时间]",
  "status": "planning",
  "writingMode": "[serial|subagent-parallel|agent-teams]",
  "coreSetting": {
    "genre": "[网文类型]",
    "goldenFinger": "[金手指类型]",
    "coreTropes": "[核心爽点]"
  },
  "chapters": [
    {
      "chapterNumber": 1,
      "title": "[章节标题]",
      "filePath": "chapters/第01章.md",
      "status": "pending",
      "wordCount": null,
      "wordCountPass": null,
      "famousScene": "[名场面类型]",
      "famousSceneDesc": "[名场面描述]",
      "emotionalArc": "[本章情绪弧线]",
      "foreshadow": {
        "seeds": ["[埋下的伏笔]"],
        "resolves": ["[回收的伏笔]"]
      },
      "retryCount": 0
    }
  ]
}
```

### JSON 状态流转

```
planning → in_progress → validating → completed
                              ↓
                           failed (可重试，最多3次)
```

## 字数检查脚本

```bash
# 检查单个章节
python scripts/check_chapter_wordcount.py ./web-novels/项目文件夹/chapters/第01章.md

# 检查所有章节
python scripts/check_chapter_wordcount.py --all ./web-novels/项目文件夹/

# 自定义最小字数
python scripts/check_chapter_wordcount.py ./web-novels/项目文件夹/chapters/第01章.md 2500
```

### 使用场景

| Phase | 用途 |
|-------|------|
| 正文撰写阶段 | 撰写后检查单章字数 |
| 润色阶段 | 润色后再次确认 |
| 校验阶段 | 批量检查所有章节字数 |

## 爽点类型速查

| 爽点类型 | 适用场景 | 关键词 |
|---------|---------|-------|
| 打脸爽 | 退婚/羞辱/质疑 | "你不是说我废物吗？" |
| 升级爽 | 突破/获得传承 | "境界突破！" |
| 装逼爽 | 被逼展示/身份反差 | "不是我想装..." |
| 甜宠爽 | 吃醋/护短/撒糖 | "以后离他远点" |
| 复仇爽 | 清算/打脸仇人 | "该还了" |
| 燃点爽 | 守护/热血/战斗 | "谁都不能动..." |

## 金句公式

```
打脸金句："你说的XX...好像有点短？"
装逼金句："不是我想装，是你们逼我的。"
燃点金句："谁都不能动我在乎的人。"
甜点金句："以后离他远点，听见没？"
```

---

## 开篇公式

```
第1章 = 觉醒爽点 + 大钩子

退婚流：废物开局 → 退婚羞辱 → 意外觉醒
系统流：濒死危机 → 系统激活 → 新手任务
重生流：死亡/失败 → 重生节点 → 先知优势
```

## 统一产物规范

### 输出目录

`./web-novels/{timestamp}-{小说名称}/`

### 必备文件

| 文件 | 作用 |
|------|------|
| `00-大纲.md` | 核心设定与章节规划 |
| `01-人物档案.md` | 主角/女主/反派信息 |
| `03-写作计划.json` | 状态机、章节进度、校验结果 |
| `chapters/第01章.md` | 章节正文，按两位数顺序命名 |

### 可选文件

| 文件 | 场景 |
|------|------|
| `02-名场面时间轴.md` | 需要提前规划爽点节奏时 |
| `04-章节详细规划.md` | Professional / Industrial 模式推荐 |
| `05-伏笔系统.md` | 需要长期管理伏笔时 |
| `QC校验报告.md` | 批量校验后生成 |
| `市场调研报告.md` | Industrial 模式可选 |

### 命名规则

- 项目目录：`{YYYYMMDD-HHmmss}-{小说名称}`
- 章节文件：`chapters/第01章.md`、`chapters/第02章.md`
- 写作计划：统一为 `03-写作计划.json`
- JSON 中 `filePath` 一律写相对路径，如 `chapters/第01章.md`

### 状态流转

`planning -> in_progress -> validating -> completed`

失败分支：`failed`

失败章节允许重试，`retryCount` 递增；校验通过后写回 `validation` 字段。

## 营销素材生成工具

> 三模式通用，可在任意 Phase 调用。用于生成书名、简介、读者评价，辅助发布准备。

### 一、书名生成

**调用时机**：Phase 1 核心明确后 / Phase 8 发布包装前

**生成原则**：
- 核心词 + 类型词 + 情绪词，三段式最优
- 必须包含关键词（系统/重生/退婚/马甲/总裁/追妻）
- 强冲突感 + 强画面感
- 8-16 字为佳

**生成公式**：

```
公式A（直白型）：[核心爽点] + [题材类型]
  例：开局即无敌、我在都市捡属性

公式B（悬念型）：[动作/状态] + [反差/悬念]
  例：废物女婿被离婚后、全球轮回

公式C（情绪型）：[情绪词] + [核心设定]
  例：甜爆！重生后我被宠上天、疯批美人

公式D（人设型）：[身份标签] + [核心冲突]
  例：假太监他权倾朝野、替身后她惊艳全球
```

**输出格式**：

```markdown
## 候选书名

| # | 书名 | 核心词 | 类型词 | 情绪词 | 适合平台 |
|---|------|--------|-------|--------|---------|
| 1 | 《XXX》 | ... | ... | ... | 起点/番茄 |
| 2 | 《XXX》 | ... | ... | ... | 起点/番茄 |
| 3 | 《XXX》 | ... | ... | ... | 起点/番茄 |

**推荐**：[书名] — 理由
```

---

### 二、作品简介生成

**调用时机**：Phase 8 发布包装 / 最终发布前

**简介结构（100-200字）**：

```
【强冲突开场】（20-30字）
  一句话建立核心矛盾，必须有冲突感

【主角身份+金手指】（30-50字）
  主角是谁 + 有什么独特优势

【核心爽点预告】（30-50字）
  暗示主要爽点类型，打脸/甜宠/升级等

【情绪钩子】（10-20字）
  一句话制造期待感或悬念

【人设标签】
  强强/甜宠/爽文/重生/马甲...（平台标签）
```

**示例**：

```markdown
## 候选简介

### 版本A（适合起点）
林逸重生都市，意外觉醒上古传承。
  — 前世窝囊废，今生都市王。
  — 校花倒追、千金倒贴、仇人跪地求饶——
  — 偏偏老婆还不满意？
【标签】都市重生 · 爽文 · 热血 · 无敌流

### 版本B（适合番茄）
苏晚穿成同名同姓的假千金，被赶出家门流落街头。
  — 等等，她怎么还有个...
  — 【豪门马甲+甜宠+打脸】
```

**违禁检查**：
- [ ] 无绝对化用词（最强/唯一/第一）
- [ ] 无色情暗示
- [ ] 无政治敏感内容
- [ ] 字数在平台要求范围内

### 三、读者评价生成

**调用时机**：Phase 8 发布包装 / 宣传素材准备

**评价类型**：

| 类型 | 用途 | 语气 |
|------|------|------|
| **种草型** | 吸引读者点击 | 真实推荐感 |
| **好奇型** | 制造悬念 | 引发讨论 |
| **打脸型** | 突出爽感 | 情绪激烈 |
| **甜宠型** | 吸引女频读者 | 姨母笑 |

**生成公式**：

```
种草型：追平了！+ 具体感受 + 行动（熬夜看/充钱了）
好奇型：没想到...+ 反转点 + 你们自己看
打脸型：看得我太爽了！+ 具体情节 + 最爱主角
甜宠型：啊啊啊！+ 甜点描述 + 磕到了
```

**输出格式**：

```markdown
## 读者评价（模拟真实读者）

### 种草型
⭐⭐⭐⭐⭐
"追平了！一口气看了200章根本停不下来，男主打脸反派那里太爽了，熬夜看完直接充了100块，值得！"
—— 读者「书虫小王」

### 好奇型
⭐⭐⭐⭐⭐
"没想到女配重生后直接逆袭了，女主居然是被穿书的？这设定太新颖了，强烈推荐！"
—— 读者「悬疑控」

### 打脸型
⭐⭐⭐⭐⭐
"看得我拳头都硬了！男主前期被踩得那么惨，后期反击的时候简直封神！"
—— 读者「热血青年」

### 甜宠型
⭐⭐⭐⭐⭐
"啊啊啊程医生太会了！这段对手戏我看了十遍，甜到姨母笑根本停不下来！"
—— 读者「小甜饼」
```

**使用场景**：
- 朋友圈/微博宣传
- 书评区置顶
- 小红书种草文案
- 读者群推广

## 营销素材使用流程

```
Phase 1 核心明确
    ↓ 生成3-5个候选书名
    ↓ 选择确认书名
Phase 8 发布包装
    ↓ 生成2-3个候选简介
    ↓ 选择优化简介
    ↓ 生成4条读者评价（种草/好奇/打脸/甜宠）
    ↓ 用于宣传素材
```
