# Phase 0 – Initialization

**触发时机**：每次 Skill 激活时首先执行

---

## 0.1 偏好加载

检查 `user-preferences.json` 是否存在：
- 存在 → 加载偏好数据（用于选项排序、推荐标记、个性化欢迎语）
- 不存在 → 使用默认值，首次交互后自动创建

偏好数据结构与更新规则详见 → [Shared_Infrastructure.md](./Shared_Infrastructure.md)

---

## 0.2 中断续写检测

扫描 `./web-novels/` 目录，查找含 `03-写作计划.json` 且 `status` 为 `"in_progress"` 或 `"validating"` 的项目。

**如有未完成项目**：
- 展示：小说名称、完成进度（X/Y 章已完成）
- 提供选项：
  - `继续上次创作` → 进入 Phase 6，从中断章节继续
  - `开始新作品` → 进入 Phase 1

**如无未完成项目** → 进入步骤 0.3

---

## 0.3 快捷入口判断

分析用户首次输入是否包含丰富的创作信息（题材、爽点、金手指等）。

**信息充足** → 展示提取结果，提供选项：
- `是的，直接开始规划` → 跳过 Phase 1-4，进入 Phase 5
- `先确认提取的信息` → 展示配置供确认
- `走一遍完整流程` → 进入 Phase 1

**信息不足** → 进入 Phase 1

---

## 0.4 目录初始化

确保 `./web-novels/` 目录存在（如不存在则创建）。

---
