# Instant 模式 - 一站式生成

**模式**: Instant 即刻模式（非交互式・单次生成）
**适用**: `claude -n -p` 非交互式调用 / 想直接看结果的快速验证

---

## 触发条件

当用户输入包含以下关键词时触发：
- "直接生成"、"一站式"、"一次性生成"
- "不需要问我"、"直接写"
- `claude -n -p` 环境调用

---

## 输入格式

用户只需提供**一句话想法**，格式不限：

```
示例：
- 都市重生文，主角是医生，有透视金手指
- 玄幻退婚流，废物男主，修炼系统
- 甜宠总裁文，女主是助理
```

---

## 生成流程

### Step 1: 解析核心要素

从用户输入中自动提取：

| 要素 | 提取规则 |
|------|---------|
| 类型 | 都市/玄幻/穿越/甜宠/悬疑等 |
| 金手指 | 系统/重生/异能/传承等 |
| 核心爽点 | 打脸/升级/甜宠/复仇等 |
| 主角设定 | 男频/女频 |
| 世界观 | 现代/古代/修仙/异能等 |

---

### Step 2: 生成完整项目

按以下结构生成全部内容：

```
项目目录：./web-novels/{timestamp}-{小说名称}/
│
├── 00-小说信息.md
│   ├── 书名：[候选书名 x3]
│   ├── 简介：[100-200字]
│   ├── 类型：[网文类型]
│   ├── 金手指：[金手指类型]
│   └── 核心爽点：[爽点列表]
│
├── 01-大纲.md
│   ├── 世界观设定
│   ├── 人物档案（主角/女主/反派）
│   ├── 阶段划分（开局/发育/高潮/结局）
│   └── 名场面时间轴
│
├── 02-章节大纲.md
│   └── 每章一句话核心事件
│
└── chapters/
    ├── 第01章.md（2000-3000字）
    ├── 第02章.md
    ├── ...
    └── 第20章.md
```

---

### Step 3: 生成规则

#### 3.1 书名生成（3个候选）

```
公式：[核心爽点词] + [类型词] + [情绪词]

示例（都市重生医生文）：
1. 《都市神医重生：开局即巅峰》
2. 《透视医圣：老婆她后悔了》
3. 《逆袭从一张CT片开始》
```

#### 3.2 简介生成

```
结构：
【强冲突开场】（20-30字）— 一句话建立核心矛盾
【主角+金手指】（30-50字）— 身份与独特优势
【核心爽点预告】（30-50字）— 主要爽点类型
【情绪钩子】（10-20字）— 制造期待感
【标签】：平台标签词

示例：
苏辰重生回到十年前，意外获得上古医道传承。
  — 前世窝囊废，今生都市医圣。
  — 校花倒追、千亿遗产、仇人跪地——
  — 唯一的问题是，他那个冰山老婆好像不太满意？
【标签】都市重生 · 医道 · 爽文 · 无敌流
```

#### 3.3 大纲生成

```
四阶段结构：
├── 开局（第1-3章）— 金手指激活/第一个冲突
├── 发育（第4-10章）— 升级打脸/情感推进
├── 高潮（第11-15章）— 大高潮/爽点密集
└── 结局（第16-20章）— 收尾/悬念/下一部伏笔

每阶段包含：
- 核心目标
- 3-5个关键情节点
- 本阶段名场面 x1
```

#### 3.4 章节正文生成

```
每章结构（2000-3000字）：
├── 开头（100-200字）— 承接上文，建立场景
├── 推进（1500-2000字）— 冲突/互动/反转
│   ├── 第一个爽点（章前1/3处）
│   ├── 核心冲突（章中1/2处）
│   └── 第二个爽点（章后1/3处）
├── 结尾（200-300字）— 留钩子/悬念/名场面
└── 金句（1句）— 让读者想截图

章内爽点密度：每章 2-3 个爽点
情绪节奏：压抑 → 释放 → 钩子
```

---

### Step 4: 读者评价生成

生成 4 条模拟读者评价（用于宣传）：

```
类型：种草型 / 好奇型 / 打脸型 / 甜宠型

格式：
⭐⭐⭐⭐⭐
"[具体感受描述]"
—— 读者「昵称」
```

## 调用示例

### Claude CLI 非交互式调用

```bash
# 基本调用
claude -n -p "生成一个都市重生文，主角是医生，有透视金手指，20章"

# 指定更多细节
claude -n -p "生成玄幻退婚流小说，男主废物开局，有修炼系统，30章，甜宠风格"

# 保存结果
claude -n -p "生成..." > output.md
```

### Python 脚本调用

```python
import subprocess

prompt = "生成一个都市爽文，主角是兵王，有透视金手指，20章"

result = subprocess.run(
    ["claude", "-n", "-p", prompt],
    capture_output=True,
    text=True
)

print(result.stdout)
```

## 生成质量保证

| 检查项 | 标准 |
|--------|------|
| 开篇抓人 | 第1章500字内有冲突/金手指 |
| 爽点密度 | 每章 2-3 个爽点 |
| 情绪节奏 | 压抑→释放，虐后必爽 |
| 金句数量 | 每章至少 1 句 |
| 钩子设置 | 每章结尾有悬念 |
| 字数控制 | 每章 2000-3000 字 |
| 逻辑一致 | 人物行为符合设定 |
| 文风统一 | 无跳跃/无矛盾 |

---

## 限制与注意事项

1. **生成速度**：20章约需 5-10 分钟（受章节数影响）
2. **内容质量**：不如 Professional 模式精细，适合快速验证想法
3. **交互能力**：本模式为纯生成模式，无交互确认
4. **建议**：生成后可导入 Professional 模式继续润色/扩展
