# 第三阶段：疯狂创作

**重要：全程无需再次向用户确认，必须逐章创作直到完成**

---

## 0. 启动检测与模式读取

开始创作前：
1. 读取 `03-写作计划.json`
2. 读取 `writingMode` 字段，进入对应模式流程
3. 如果存在 `status: "in_progress"` 的章节 → 从该章节继续（中断续写）
4. 如果所有章节 `status: "pending"` → 从第 1 章开始
5. 如果存在 `status: "failed"` 的章节（Phase 4 回退）→ 从第一个 failed 章节开始

---

## 1. 逐章创作流程（通用，所有模式共用）

每章创作时严格执行以下步骤：

### 步骤 1: 写前分析（必须执行）

1. 读取 `03-写作计划.json` — 查看各章节状态，确定下一个待创作章节
2. **读取 `00-大纲.md`** — 找到当前章节的规划信息，提取：
   - 核心事件
   - 爽点设计
   - 承接上章
   - 悬念钩子
   - 名场面类型（如有）
   - 伏笔埋设与回收（如本章有）
3. **一致性检查** — 如果不是第1章：
   - 读取上一章的章节摘要
   - 核对主角状态、能力值、关系进展是否衔接
   - 检查待回收的伏笔是否已在之前章节埋下
4. 更新 `03-写作计划.json` — 将本章 `status` 设为 `"in_progress"`

### 步骤 2: 撰写

5. 创建章节文件 — 使用 [chapter-template.md](../guides/chapter-template.md) 模板
6. **基于大纲规划创作** — 严格按照大纲中本章的核心事件和爽点设计撰写正文
7. 撰写正文 — **每章必须达到 2000-3000 字**
   - **情绪曲线**：必须有压抑→释放的弧线（憋→爽）
   - **人物语气**：对话需符合人物性格和当前处境
   - 开头检查：前 20% 是否有即时爽点？→ [网文开篇公式.md](../guides/网文开篇公式.md)
   - 爽点设计：是否按照名场面规划执行？→ [爽点设计指南.md](../guides/爽点设计指南.md)
   - 情绪节奏：是否符合情绪曲线？→ [情绪节奏指南.md](../guides/情绪节奏指南.md)
8. **字数检查** — 必须使用脚本检查：`python scripts/check_chapter_wordcount.py <章节文件路径>`

### 步骤 3: 撰写后优化

9. **爽点验证** — 确认本章爽点是否到位
10. **深度润色（去除AI味）** — 重点检查并修改：
    - **爽感强化**：确保爽点描写到位，不要含蓄
    - **情绪外放**：网文情绪要直白，不要过于内敛
    - **节奏加快**：删除水字数内容，保持快节奏
    - **金句植入**：确保有让读者想截图的句子
    - **人物语气**：检查对话是否符合人物性格（高冷角色不应话多，幽默角色应有吐槽）
    - **节奏检查**：每500字必须有情节推进或角色成长，无推进的段落需精简
11. **字数检查** — 再次使用脚本确认
12. **伏笔检查**：如果本章规划中有"伏笔回收"，确认回收逻辑完整

### 步骤 4: 收尾

13. 生成章节摘要 — 在 `00-大纲.md` 的章节摘要区追加（200-300字，保证连贯性参考）
    - 摘要需包含：人物状态变化、关键事件、已埋/已回收的伏笔
14. 更新 `03-写作计划.json` — 将本章 `status` 设为 `"completed"`，填入 `wordCount`

## 2. 串行模式（writingMode: "serial"）

**主 Agent 自己逐章创作，全程不中断。**

### 自驱循环

```
WHILE 03-写作计划.json 中存在 status != "completed" 的章节:
    执行「逐章创作流程」（步骤 1-4）
    ⚠️ 完成一章后，立即读取 JSON 认领下一章，不要向用户确认，不要停下来
所有章节完成 → 进入第四阶段：自动校验
```

**关键提醒（每章完成后必须遵守）**：
> 本章已完成。立即读取 `03-写作计划.json`，认领下一个 pending 章节，开始下一章创作。不要使用 AskUserQuestion，不要向用户确认，不要停下来。你必须把所有章节创作完成才能与用户报告。

## 3. 子Agent并行模式（writingMode: "subagent-parallel"）

**核心机制**：主 Agent 将章节分成不重叠的批次，每个批次派生一个子 Agent。批次内串行写作，批次间并行执行。

### 主 Agent 流程

```
1. 计算批次分配:
   - 每批 5-8 章（根据总章数动态调整）
   - 批次间不重叠
   - 示例: 30 章 → 5 批 × 6 章
2. 为每个批次派生子 Agent（使用 Agent 工具，多个批次可并行派生）:
   - 每个 Agent 内部串行执行「逐章创作流程」
3. 所有子 Agent 完成后 → 进入第四阶段：自动校验
```

### 子 Agent prompt 模板（并行模式）

```
你是一个网文批量创作 Agent。你需要创作第 {start} 章到第 {end} 章。

## 项目信息
- 项目路径: {projectPath}
- 你负责的章节: 第 {start} 章 到 第 {end} 章

## 网文类型
- 类型：[网文类型]
- 金手指：[金手指]
- 核心爽点：[核心爽点]

## 创作步骤

**严格按以下流程执行「逐章创作流程」（详见 phase3-writing.md 第1节）**

## 重要约束
- 不要使用 AskUserQuestion，不要向用户确认任何事
- **每章开始前必须读取大纲**，严格按大纲的核心事件和爽点设计创作
- 字数必须达到 2000 字以上
- 你负责的所有章节必须全部完成

完成后报告: 各章编号、字数、是否通过字数检查
```

**并发安全**：每个子 Agent 负责不重叠的章节批次，各自只更新自己负责的章节状态，不存在写入冲突。

## 4. Agent Teams 模式（writingMode: "agent-teams"）

**Claude Code 特有模式**。需用户手动开启。通过 TeamCreate 创建写作团队，Agent 之间可以通讯协作。

### 主 Agent 流程

```
1. 使用 TeamCreate 创建写作团队
2. 派生 3-5 个团队成员（使用 Agent 工具 + team_name 参数）
3. 使用 TaskCreate 为每一章创建一个任务
4. 团队成员各自:
   a. 从 TaskList 读取可用任务
   b. 使用 TaskUpdate 认领任务（设置 owner）
   c. 执行「逐章创作流程」（见第1节）
   d. 使用 TaskUpdate 标记任务完成
   e. 回到 TaskList 认领下一个任务
5. 所有任务完成后 → 关闭团队 → 进入第四阶段：自动校验
```

### 团队成员 prompt 模板

```
你是一个网文小说创作团队成员。你的任务是创作分配给你的章节。

## 项目信息
- 项目路径: {projectPath}
- 类型：[网文类型]
- 金手指：[金手指]

## 工作流程

1. 从 TaskList 读取可用任务
2. 使用 TaskUpdate 认领任务（设置 owner 为你的名字）
3. **首先读取 {projectPath}/00-大纲.md**，找到该章节的规划信息
4. 读取 03-写作计划.json，获取该章节状态
5. **执行「逐章创作流程」**（详见 phase3-writing.md 第1节）
6. 使用 TaskUpdate 标记任务完成
7. 回到 TaskList 认领下一个任务

## 重要约束
- 不要使用 AskUserQuestion，不要向用户确认任何事
- **每章开始前必须读取大纲**，严格按大纲的核心爽点设计创作
- 字数必须达到 2000 字以上
- 与其他团队成员通过 SendMessage 协调，避免写同一章节
```

**并发安全**：通过 TaskList 系统的内置所有权语义（owner 字段）和 Agent 间通讯避免冲突。
